Retrieval-Augmented Generation, commonly abbreviated as RAG, is a design pattern for building AI applications that answer questions using a specific set of documents rather than relying purely on what a language model already knows. RAG works in three steps: retrieve relevant information from a knowledge base, augment the model's prompt with that information as context, and then have the model generate an answer grounded in that context.

The main benefit of RAG is that it reduces hallucinations, which happen when a language model confidently generates an answer that sounds plausible but is actually incorrect. By supplying the model with real, relevant text from a trusted document set, RAG makes it much more likely that the model's answer will be accurate and traceable back to a specific source.

A typical RAG pipeline begins with data ingestion, where source documents are split into smaller chunks, usually at the paragraph level, since retrieving very large documents wholesale is inefficient and can exceed a model's context window. Each chunk is then converted into an embedding and stored, often alongside the original document text, so that both the vector and the source text are available later.

At query time, the user's question is embedded and compared against the stored chunk embeddings to find the most relevant pieces of text. Those chunks are inserted into the prompt sent to the language model, usually alongside instructions telling the model to answer only using the provided context and to avoid guessing when the answer is not present in the retrieved text.
